DraftKings Opponent Finder Chrome Extension: Sprint 3 User Stories

Sprint 3: Username Blacklist Feature

Sprint Goal: Implement a Username Blacklist feature that allows users to mark contests containing specific opponents and manage their blacklist.

User Story 9: Blacklist Management UI

As a user,
I want to be able to add and remove usernames from my blacklist,
So that I can avoid contests with specific opponents.

Tasks:

9.1 Design a user interface for managing the blacklist within the extension popup
9.2 Implement an input field for adding new usernames to the blacklist
9.3 Create a list view of currently blacklisted usernames
9.4 Add functionality to remove usernames from the blacklist
9.5 Implement input validation to prevent duplicate or invalid entries

Acceptance Criteria:

  • Users can add new usernames to the blacklist
  • Users can view all currently blacklisted usernames
  • Users can remove usernames from the blacklist
  • The UI prevents addition of duplicate or invalid usernames
  • The blacklist management interface is intuitive and easy to use

User Story 10: Blacklist Storage

As a user,
I want my blacklist to persist between browsing sessions,
So that I don't have to recreate it every time I use the extension.

Tasks:

10.1 Implement a storage mechanism for the blacklist using Chrome's storage API
10.2 Create functions to save the blacklist when it's updated
10.3 Implement functionality to load the blacklist when the extension is initialized
10.4 Add error handling for storage operations

Acceptance Criteria:

  • Blacklist persists between browser sessions
  • Changes to the blacklist are immediately saved
  • Blacklist is loaded correctly when the extension is opened
  • Storage errors are handled gracefully without crashing the extension

User Story 11: Integrate Blacklist with Contest Rating

As a user,
I want contests containing blacklisted opponents to be clearly marked,
So that I can easily avoid these contests.

Tasks:

11.1 Modify the contest parsing function to check for blacklisted usernames
11.2 Update the rating calculation to set the rating to "X" when a blacklisted user is found
11.3 Implement a visual indicator (e.g., different color or icon) for contests with blacklisted users
11.4 Update the hover information to include the reason for the "X" rating
11.5 Ensure the blacklist check doesn't significantly impact performance

Acceptance Criteria:

  • Contests with blacklisted opponents are correctly identified
  • These contests are assigned an "X" rating
  • Contests with blacklisted opponents have a distinct visual indicator
  • Hover information explains why a contest is marked with an "X"
  • Blacklist integration doesn't noticeably slow down the extension

User Story 12: Blacklist Feature Documentation

As a user,
I want clear instructions on how to use the blacklist feature,
So that I can fully utilize this functionality.

Tasks:

12.1 Write user documentation explaining the blacklist feature
12.2 Create a brief tutorial or walkthrough for first-time users
12.3 Update the extension's help section to include blacklist information
12.4 Add tooltips or hints in the UI to guide users

Acceptance Criteria:

  • Clear and concise documentation for the blacklist feature is available
  • A tutorial or walkthrough guides new users through the blacklist functionality
  • The extension's help section includes up-to-date information about the blacklist
  • UI elements have helpful tooltips or hints for user guidance

Sprint 3 Deliverables

  • Functional Username Blacklist management interface
  • Persistent storage for the blacklist
  • Integration of blacklist with contest rating system
  • Visual indicators for contests with blacklisted opponents
  • Updated documentation and user guidance for the blacklist feature
