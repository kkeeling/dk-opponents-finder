DraftKings Opponent Finder Chrome Extension: Sprint 1 User Stories

Sprint 1: Project Setup and Core Functionality

Sprint Goal: Set up the project infrastructure and implement basic functionality for scanning the DraftKings lobby and fetching contest details.

User Story 1: Project Setup

As a developer,
I want to set up the initial project structure and development environment,
So that I can start building the Chrome extension efficiently.

Tasks:

1.1.1 Create a new directory for the project
1.1.2 Initialize a Git repository
1.1.3 Set up a Node.js project with npm
1.1.4 Install necessary development dependencies (e.g., webpack, babel)
1.1.5 Configure ESLint for code linting
1.1.6 Set up a basic folder structure for the extension

Acceptance Criteria:

  • Project directory is created with a proper structure
  • Git repository is initialized with a .gitignore file
  • package.json file is created with necessary dependencies
  • ESLint is configured and functional
  • Developer can run basic npm commands (start, build, lint)

User Story 2: Extension Manifest

As a developer,
I want to create the manifest file for the Chrome extension,
So that the browser can recognize and load the extension properly.

Tasks:

1.2.1 Create a manifest.json file in the project root
1.2.2 Define basic extension metadata (name, version, description)
1.2.3 Specify necessary permissions for DraftKings website access
1.2.4 Define content scripts and their matches
1.2.5 Add placeholder icons for the extension

Acceptance Criteria:

  • manifest.json file is created and valid
  • Extension can be loaded in Chrome's developer mode
  • Correct permissions are set for DraftKings website
  • Content scripts are properly defined and matched to DraftKings URLs

User Story 3: Lobby Page Scanning

As a user,
I want the extension to scan the DraftKings lobby page,
So that it can identify contests for further analysis.

Tasks:

1.3.1 Create a content script file (content.js)
1.3.2 Implement a function to detect when the user is on the DraftKings lobby page
1.3.3 Develop logic to scan the page for contest rows
1.3.4 Extract contest IDs and entry counts from each row
1.3.5 Implement basic error handling for the scanning process

Acceptance Criteria:

  • Content script successfully injects into the DraftKings lobby page
  • Script can identify and extract information from contest rows
  • Contest IDs and entry counts are correctly parsed
  • Console logs or debug output show successful scanning of the lobby page

User Story 4: Fetching Contest Details

As a user,
I want the extension to fetch details for each identified contest,
So that I can later analyze the opponent information.

Tasks:

1.4.1 Implement a function to construct the contest detail URL
1.4.2 Develop a mechanism to fetch the HTML content of the contest detail page
1.4.3 Implement request throttling to avoid overwhelming DraftKings' servers
1.4.4 Create a basic caching system to store fetched contest details
1.4.5 Implement error handling for failed requests

Acceptance Criteria:

  • Extension can successfully fetch contest detail pages
  • Requests are throttled to respect DraftKings' servers
  • Fetched data is temporarily cached to minimize redundant requests
  • Error handling gracefully manages failed requests
  • Console logs or debug output show successful fetching of contest details

User Story 5: Basic Testing Setup

As a developer,
I want to set up a basic testing framework,
So that I can ensure the quality and reliability of the extension's core functionality.

Tasks:

1.5.1 Choose and install a testing framework (e.g., Jest)
1.5.2 Set up test configuration files
1.5.3 Write basic unit tests for lobby scanning functions
1.5.4 Write basic unit tests for contest detail fetching functions
1.5.5 Implement a simple CI process to run tests automatically

Acceptance Criteria:

  • Testing framework is installed and configured
  • Basic unit tests are implemented for core functions
  • All implemented tests pass successfully
  • CI process is set up and runs tests on each commit

Sprint 1 Deliverables

  • Fully set up development environment
  • Functional Chrome extension with basic manifest
  • Content script that can scan the DraftKings lobby page
  • Functionality to fetch contest details
  • Basic testing suite for core functions
